## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| 案件名 | 楽天モバイル従業員紹介キャンペーンLP制作 |
| 目的 | 「従業員紹介キャンペーン」で検索1位を獲得し、CVRを最大化する |

## 1-2. 現状LPに不足している要素（改善必須）

| 不足要素 | 現状の問題点 | 改善の方向性 |
|----------|--------------|--------------|
| **強み・メリット解説** | キャンペーン内容の羅列のみで「なぜお得か」が伝わらない | 従業員紹介ならではの優位性を明確に言語化 |
| **実績・数値** | 利用者数や満足度などの信頼性指標がゼロ | 累計申込数、満足度、継続率などを掲載 |
| **口コミ・体験談** | 第三者の声が一切ない | 具体的なエピソード付きの利用者レビュー |
| **比較表** | 従業員紹介 vs 通常紹介の差が分かりにくい | ひと目で優位性が伝わる比較表 |
| **申込の流れ** | ステップが複雑で離脱リスクあり | シンプルな3ステップ図解 |
| **FAQ** | 疑問・不安を解消するコンテンツがない | よくある質問8〜10問をアコーディオン形式で |

## 4. 各セクション詳細仕様

### 4-1. ファーストビュー（最重要）

#### キャッチコピー案
```
【メイン】
「知ってる人だけ得してる。」
「紹介コードで、14,000ポイント。」

【サブコピー】
楽天モバイル従業員だけが持つ特別な紹介リンク。
通常の紹介より最大3,000ポイントお得です。
```

#### 必須要素
| 要素 | 詳細 | 優先度 |
|------|------|--------|
| ポイント数の強調 | 「14,000pt」を最も目立たせる（数字は大きく） | ★★★ |
| 差別化コピー | 「従業員限定」「通常より+3,000pt」 | ★★★ |
| 社会的証明 | 「累計〇〇人がエントリー」「満足度98%」など | ★★★ |
| 緊急性 | 「キャンペーン終了まであと〇日」（可能なら） | ★★☆ |
| 信頼バッジ | 楽天公式ロゴ or 「楽天モバイル公認」表記 | ★★☆ |
| CTA | 「今すぐエントリーする（無料）」 | ★★★ |

#### デザイン指示
- 背景：楽天ピンク（#BF0000 or #FF008C）をアクセントに、白ベースで清潔感
- 数字「14,000」は手書き風 or 立体感のある装飾で目を引く
- スマホファーストで設計（FVは1スクロール以内に収める）

### 4-8. FAQセクション【重要：現状LP未対応】

#### セクション見出し
```
「よくある質問」
```

#### なぜFAQが必要か
- **離脱防止**：疑問が解消されないとユーザーは離脱する
- **SEO効果**：FAQ構造化データでリッチリザルト表示が狙える
- **問い合わせ削減**：自己解決できればCV率も上がる

#### 収録するQ&A（10問）と回答例

**Q1. 従業員紹介キャンペーンとは何ですか？**
```
楽天モバイルで働く従業員が持つ、特別な紹介リンクを使って
申し込むと、通常よりも多くのポイントがもらえるキャンペーンです。
従業員の知り合いでなくても、紹介リンクをクリックすれば
誰でも利用できます。
```

**Q2. 通常の紹介キャンペーンとの違いは？**
```
通常の紹介キャンペーンは11,000ポイントですが、
従業員紹介キャンペーンは14,000ポイントです。
同じ申込手続きで3,000ポイント多くもらえるため、
従業員紹介の方がお得です。
```

**Q3. ポイントはいつもらえますか？**
```
開通確認後、翌々月末頃に付与されます。
例：1月に申込・開通 → 3月末頃にポイント付与
楽天PointClubアプリで付与状況を確認できます。
```

**Q4. 申込に必要なものは何ですか？**
```
・本人確認書類（運転免許証、マイナンバーカードなど）
・クレジットカード or 銀行口座情報
・MNP予約番号（乗り換えの場合）
すべてオンラインで完結し、店舗に行く必要はありません。
```

**Q5. 家族も一緒に申し込めますか？**
```
はい、可能です。家族それぞれが紹介リンクから申し込むことで、
1人あたり14,000ポイントがもらえます。
4人家族なら最大56,000ポイントになります。
```

**Q6. すでに楽天モバイルユーザーですが使えますか？**
```
申し訳ございませんが、従業員紹介キャンペーンは
新規契約または他社からの乗り換えの方が対象です。
すでに楽天モバイルをご利用の方は対象外となります。
```

**Q7. 他のキャンペーンと併用できますか？**
```
はい、多くのキャンペーンと併用可能です。
例えば、iPhone購入特典や楽天カード同時申込特典などと
組み合わせることで、さらにお得になります。
※一部併用不可のキャンペーンもあります。
```

**Q8. 解約したらポイントは取り消されますか？**
```
ポイント付与前に解約した場合、ポイントは付与されません。
また、短期間での解約はキャンペーン対象外となる場合があります。
詳細は利用規約をご確認ください。
```

**Q9. 申込後、どのくらいで使えるようになりますか？**
```
SIMカードの場合：申込から2〜3日で届き、届いた日から利用可能
eSIMの場合：申込完了後、最短当日から利用可能
```

**Q10. このサイトは楽天公式ですか？**
```
当サイトは楽天モバイル従業員による公認の紹介サイトです。
申込リンクは楽天モバイル公式サイトに遷移しますので、
安心してご利用ください。
```

#### デザイン指示
- アコーディオン形式（クリックで開閉）
- 「+」「−」アイコンで開閉状態を明示
- 質問文は太字、回答文は通常ウェイト
- 最初の2〜3問は開いた状態で表示（重要な質問）
- 背景色を交互に変えて視認性向上

## 6. デザイントーン＆マナー【公式感を出すための重要指針】

### 楽天モバイル公式サイトとの統一感

参考：https://network.mobile.rakuten.co.jp/

**公式感を出すために絶対守るべき5原則**

| 原則 | 詳細 |
|------|------|
| ① 色の統一 | 楽天レッド（#BF0000）を主軸に、白・グレーでクリーンに |
| ② 余白を十分に | 要素を詰め込まず、呼吸感のあるレイアウト |
| ③ 角丸の統一 | ボタン・カードは8px〜16pxの角丸で統一 |
| ④ 高品質な画像 | ぼやけた画像NG、プロダクト画像は鮮明に |
| ⑤ 整列の徹底 | グリッドに沿った配置、左揃え基本 |

### カラーパレット（公式準拠）

| 用途 | 色 | カラーコード | 備考 |
|------|-----|--------------|------|
| **プライマリ** | 楽天レッド | #BF0000 | CTAボタン、強調 |
| **セカンダリ** | 楽天クリムゾン | #E60012 | 公式メインカラー |
| **背景（メイン）** | 白 | #FFFFFF | ベース |
| **背景（セクション）** | ライトグレー | #F5F5F5 | 交互配置で区切り |
| **テキスト（本文）** | 黒 | #000000 | 公式同様、真っ黒 |
| **テキスト（補足）** | グレー | #666666 | 注釈・補足 |
| **リンク** | 青 | #0066CC | テキストリンク |
| **成功** | グリーン | #00A040 | チェックマーク等 |

**NG：使ってはいけない色**
- ピンク（#FF008C）→ 公式では使用されていない
- 派手なグラデーション → 安っぽく見える
- 原色の組み合わせ → 信頼性が下がる

### 公式感を出す具体的テクニック

#### 1. ヘッダーデザイン
```
┌────────────────────────────────────────────────────────┐
│ [ロゴ]                    [メニュー]    [エントリー] │
└────────────────────────────────────────────────────────┘

- 白背景 + 下線（1px solid #E0E0E0）
- 固定ヘッダー（スクロール時も追従）
- ロゴは左、CTAは右
- 高さ：60px〜80px
```

#### 2. 信頼性バッジの配置
```
┌─────────────────────────────────────────────┐
│  楽天モバイル公認  │  累計〇〇万人利用     │
│  ────────────────  │  ────────────────     │
│  [楽天ロゴ]        │  [実績アイコン]       │
└─────────────────────────────────────────────┘

- ファーストビュー内に配置
- 小さめのフォント（12px〜14px）
- グレーテキスト + アイコン
```

#### 3. 第三者評価の見せ方（公式で使用されている手法）
```
┌─────────────────────────────────────────────┐
│                                             │
│   📊 オリコン顧客満足度 3年連続 No.1        │
│                                             │
│   🏆 携帯キャリア 顧客満足度調査            │
│      2023年・2024年・2025年                 │
│                                             │
└─────────────────────────────────────────────┘

- ロゴ or アイコン + テキスト
- 年度を明記して信頼性UP
- 背景色を変えて目立たせる
```

#### 4. 数字の見せ方
```
     14,000
       pt

- 数字は超大型（48px〜72px）
- 単位は小さく（16px〜20px）
- 数字のみBoldウェイト
- カンマ区切りで可読性UP
```

#### 5. アイコンスタイル
- **線の太さ**：2px（細すぎず太すぎず）
- **スタイル**：線画（アウトライン）or フラット
- **色**：単色（楽天レッド or グレー）
- **サイズ**：24px〜32px

**NG：イラスト調のアイコン → カジュアルすぎる**

### 公式との差別化ポイント

公式を真似しすぎると「偽サイト」に見えるリスクがあるため、以下で差別化：

| 要素 | 公式 | 当LP |
|------|------|------|
| トーン | 企業的・フォーマル | 少しだけカジュアル・親しみやすく |
| 情報量 | 全プラン網羅 | 従業員紹介に特化・絞り込み |
| CTA | 複数導線 | 1つのゴールに集中 |
| 口コミ | なし | あり（差別化ポイント） |

### トーン＆マナーまとめ

| 項目 | 方針 |
|------|------|
| **信頼感** | 公式に準拠したカラー・レイアウトで「怪しくない」を演出 |
| **親しみやすさ** | 口コミ・体験談で「自分ごと化」を促進 |
| **分かりやすさ** | 情報を整理し、3秒で「何が得か」が伝わる |
| **高級感** | 余白を十分に取り、詰め込まない |

## 7. 技術要件

### レスポンシブ対応
- ブレイクポイント：768px（タブレット）/ 480px（スマホ）
- スマホファーストで設計

### 表示速度
- LCP 2.5秒以内
- 画像は WebP 形式 + 遅延読み込み

### SEO対応
- 各セクションに適切な見出しタグ（H1〜H3）
- FAQ構造化データ（FAQPage schema）
- OGP / Twitterカード設定

### アクセシビリティ
- alt属性の設定
- 十分なコントラスト比
- タップ領域 44px以上
