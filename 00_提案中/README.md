# 00_提案中（商談前・提案後の管理）

## 目的
- 商談前〜提案後〜受注/失注までの状況を、案件単位で記録する
- 受注後は該当クライアント配下へ移管し、提案内容・見積根拠を残す

## ステータス定義（例）
- **商談前**: 問い合わせ/紹介〜初回ヒアリング前
- **商談中**: ヒアリング中〜提案準備中
- **提案後**: 提案提出済み〜意思決定待ち
- **受注**: 契約/発注確定（→ `クライアント/<社名>/` へ移管）
- **失注**: 見送り確定（学びを残す）

## 案件一覧（テンプレ）
| 会社名 | 案件名 | ステータス | 期待KPI | 見積（月額/初期） | 次アクション | 期限 | 担当 |
|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |

## 記録ルール
- 提案資料や要件メモは、できるだけこのREADMEに要点を残す（ファイルを別途置く場合も要点は残す）
- 受注になったら、以下をクライアントフォルダにコピー/要約して移管する
  - 目的/KPI
  - 施策方針（優先度）
  - 見積（売上）と想定工数（原価/経費の根拠）
