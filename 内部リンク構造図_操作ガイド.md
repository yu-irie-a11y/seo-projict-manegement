# 内部リンク構造図 — 操作ガイド（チームメンバー向け）

このガイドでは、内部リンク構造図（グラフビュー）を使って **被リンクの分布を把握し、改善対象を特定する** 方法を説明します。

---

## 1. 画面の見方

### 1-1. 全体レイアウト

```
┌─────────────────────────────────────────────────────────┐
│ [ヘッダー] タイトル / 検索ボックス / ビュー切替 / 統計  │
├────────┬──────────────────────────────────┬──────────────┤
│        │                                  │              │
│セクション│       グラフビュー               │  分析パネル  │
│フィルター│    （ノード＋リンク線）          │   または     │
│ (左)   │                                  │  詳細パネル  │
│        │                                  │   (右)       │
│        │                                  │              │
├────────┴──────────────────────────────────┴──────────────┤
│                 [ズームコントロール]                      │
└─────────────────────────────────────────────────────────┘
```

### 1-2. グラフの要素

| 要素 | 意味 |
|---|---|
| **丸いノード（実線の枠）** | 解析対象としてクロールしたページ |
| **丸いノード（点線の枠）** | リンク先として検出されたが、解析対象に含まれていないページ（カテゴリページ等） |
| **ノードの大きさ** | 被リンク数が多いほど大きい |
| **ノードの色** | URLパスのセクション別（同じ色 = 同じセクション） |
| **矢印付きの線** | 内部リンクの方向（矢印の先がリンク先） |

## 2. 基本操作

### 2-1. 画面の移動とズーム

| 操作（トラックパッド） | 効果 |
|---|---|
| **2本指でスクロール** | 画面をパン（上下左右に移動） |
| **ピンチイン/アウト** | ズームイン/ズームアウト |
| **画面下部の +/- ボタン** | ズームイン/ズームアウト |
| **「全体表示」ボタン** | グラフ全体が画面に収まるようにフィット |

| 操作（マウス） | 効果 |
|---|---|
| **ドラッグ（背景）** | 画面をパン |
| **Ctrl + スクロール** | ズームイン/ズームアウト |

### 2-2. ノードの操作

| 操作 | 効果 |
|---|---|
| **マウスホバー** | ツールチップ表示（ページ名、パス、被リンク数、発リンク数） |
| **ノードをクリック** | そのノードの詳細パネルを表示＋接続ノードをハイライト |
| **ノードをドラッグ** | ノード位置を手動で移動（配置を微調整したいとき） |
| **背景をクリック** | ハイライト解除＋分析パネルに戻る |

## 3. 検索で特定のページを探す

### 3-1. 基本的な使い方

1. ヘッダーの**検索ボックス**にページ名やURLの一部を入力
2. ドロップダウンに候補が表示される（被リンク/発リンク数付き）
3. クリックまたは矢印キー（↑↓）で選択 → **Enter**で即座にズーム

### 3-2. 検索のコツ

| 入力例 | 用途 |
|---|---|
| `SEO` | SEO関連の記事を一覧 |
| `owned-media` | URLパスにowned-mediaを含む記事を絞り込み |
| `内部リンク` | タイトルに「内部リンク」が含まれる記事 |

**ポイント**: 検索結果には`IN:3 OUT:12`のように被リンク/発リンク数が表示されるので、被リンクが少ない記事をすぐに見つけられる。

### 3-3. キーボードショートカット

| キー | 効果 |
|---|---|
| **↑ / ↓** | 検索結果の候補を移動 |
| **Enter** | 選択中のノードにズーム＋ハイライト |
| **Escape** | 検索結果を閉じて元の表示に戻る |

## 4. 分析パネルで被リンク分布を把握する

### 4-1. サマリーカード

画面右側の分析パネル上部にある**4色のサマリー**が、全体の被リンク分布を一目で示す。

| 色 | 範囲 | 意味 |
|---|---|---|
| **赤** | 0件 | 被リンクゼロ。孤立状態で最優先の改善対象 |
| **橙** | 1〜3件 | 被リンクが少ない。改善の余地が大きい |
| **青** | 4〜10件 | 一定の被リンクあり。状況によっては追加を検討 |
| **緑** | 11件以上 | 十分な被リンク。基本的にOK |

### 4-2. 全ページリスト（被リンク数昇順）

サマリーの下に、**全ページが被リンク数の少ない順**で並んでいる。

```
── 0件 (8ページ) ═══════════════════
  美容クリニックSEO対策                  ↗  0
  コンテンツマーケティング外注            ↗  0
  ...

── 1件 (12ページ) ══════════════════
  代替テキストSEO効果                    ↗  1
  指名検索SEO                            ↗  1
  ...

── 2件 (15ページ) ══════════════════
  ...
```

**各行の操作:**
- **行をクリック** → グラフ上でそのノードにズーム（被リンク元/先がハイライトされる）
- **↗をクリック** → そのページの実URLを別タブで開く

### 4-3. 分析パネルの活用フロー

```
Step 1: サマリーの赤（0件）と橙（1〜3件）の数を確認
          → 「改善対象がどのくらいあるか」を把握

Step 2: リストをスクロールして0件→1件→2件...と順に確認
          → 各ページの内容を↗で確認しながら
            「どの記事からリンクを貼れるか」を検討

Step 3: 気になるページの行をクリック
          → グラフ上で接続状況を確認
          → 近くにあるが繋がっていないノードに注目

Step 4: 改善アクションをリストアップ
          → 「記事Aの○○セクションに記事Bへの内部リンクを追加」
```

## 5. 詳細パネルで個別ページの接続状況を見る

### 5-1. 詳細パネルの開き方

グラフ上の任意のノードをクリックすると、右側に**詳細パネル**が表示される。

### 5-2. パネルの構成

```
┌──────────────────────────┐
│ ページタイトル            │
│ /web-school/xxx/          │
│ [このページを開く ↗]     │
│                          │
│ 被リンク 12  発リンク 8   │
│                          │
│ ── 発リンク（8件）──     │
│ "SEO対策" → キーワード選定  ↗ ▶│
│ "詳しくは" → 内部リンクSEO  ↗ ▶│
│ ...                      │
│                          │
│ ── 被リンク（12件）──    │
│ "関連記事" ← コンテンツSEO  ↗ ▶│
│ ...                      │
└──────────────────────────┘
```

### 5-3. 各要素の操作

| 要素 | 操作 | 効果 |
|---|---|---|
| **「このページを開く ↗」** | クリック | 実際のWebページを別タブで開く |
| **リンク行のテキスト部分** | クリック | そのリンク先/元のノードにグラフ上でズーム |
| **↗（各行右側）** | クリック | そのリンク先/元の実URLを別タブで開く |
| **▶（各行右端）** | — | クリック可能であることを示す矢印 |
| **×（右上）** | クリック | 詳細パネルを閉じて分析パネルに戻る |

### 5-4. 詳細パネルの活用例

**例: 「SEO記事の書き方」の被リンクが2件しかない場合**

1. 分析パネルで「SEO記事の書き方」をクリック → グラフにズーム
2. 詳細パネルで被リンク元を確認 → 「キーワード選定」と「SEO入門」の2記事からのみ
3. グラフ上で近くにあるが線が繋がっていないノードを確認
4. 「コンテンツSEO」「SEO内部対策」等の関連記事を発見
5. → これらの記事から「SEO記事の書き方」への内部リンクを追加する施策を立案

## 6. ヒートマップモード

### 6-1. 切り替え方

ヘッダーの**「被リンク ヒートマップ」**ボタンをクリック。

### 6-2. 見方

ノードの色が被リンク数に応じたグラデーションに変わる。

| 色 | 意味 |
|---|---|
| **赤〜オレンジ** | 被リンクが少ない（要改善） |
| **青〜緑** | 被リンクが多い（良好） |

### 6-3. 使いどころ

- **全体俯瞰**: グラフを引きで見たとき、赤いエリアが集中していれば「このセクション全体で被リンクが弱い」と分かる
- **グラフモードに戻す**: 「グラフ」ボタンをクリック

---

## 7. セクションフィルター

### 7-1. 使い方

左パネルの**セクション名**をクリックすると、そのセクションのノードの表示/非表示を切り替えられる。

### 7-2. 活用例

- 共通リンク先（`/contact/`, `/useful/`等）が画面を圧迫しているとき → これらのセクションを非表示にして本文リンクに集中
- 特定セクション（例: `web-school`）だけに絞って、セクション内の内部リンク状況を確認

## 8. よくある分析パターン

### パターン1: 被リンクゼロの記事を見つけて改善する

```
分析パネル → 0件のリストを確認
  → 記事の内容を↗で確認
  → 関連する記事を検索ボックスで探す
  → 関連記事の発リンク一覧を見て「この記事に追加できそうか」判断
  → 施策: 「記事Xの○○部分に記事Yへの内部リンクを追加」
```

### パターン2: 特定の記事への内部リンクが偏っていないか確認

```
検索ボックスで対象記事を検索 → クリック
  → 詳細パネルで被リンク元を一覧確認
  → 「全部同じセクションからしか被リンクがない」等の偏りを発見
  → 他セクションの関連記事から横断的にリンクを追加
```

### パターン3: トピッククラスターの接続状況を確認

```
検索ボックスで「SEO」等のキーワードで検索
  → 検索結果一覧でSEO関連記事のIN/OUT数を比較
  → 被リンクが少ない記事をクリック → グラフでクラスター内の接続を確認
  → クラスター内の記事同士が相互リンクされているか目視チェック
```

### パターン4: 新規記事のリンク設計

```
新記事を公開した後に再生成
  → 分析パネルで被リンク0件として表示される
  → グラフ上で関連テーマの記事群を探す
  → 既存記事から新記事へ、新記事から既存記事への双方向リンクを設計
```

## 9. 用語集

| 用語 | 意味 |
|---|---|
| **被リンク（IN）** | 他のページ**から**このページへのリンク。多いほどSEO評価が集まりやすい |
| **発リンク（OUT）** | このページ**から**他のページへのリンク。関連ページへの導線 |
| **孤立ページ** | 被リンクが0〜1件のページ。検索エンジンがたどり着きにくい |
| **トピッククラスター** | 共通テーマの記事群が相互にリンクし合う構造。SEOに有効 |
| **リンクジュース** | 被リンクを通じて伝搬するSEO評価。内部リンクで意図的に配分できる |
| **ノード** | グラフ上の丸（= 1ページ） |
| **エッジ** | グラフ上の線（= 1つの内部リンク） |
| **セクション** | URLパスから自動推定されるページのグループ分け |
